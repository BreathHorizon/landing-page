第一章 定义与解释

  • 灵息 AI/本平台：指为用户提供多模型聚合访问的工具与服务平台（网站、客户端、接口等）。
  • 第三方模型/供应商：指由 OpenAI、Google DeepMind、Anthropic、DeepSeek、阿里云、百度、智谱 AI 等第三方提供的模型与服务。
  • 普通用户（免费版）：指注册后未开通 Plus 的用户，仅可通过平台界面进行交互，不提供任何 API 权限。
  • Plus 用户：指通过登记审核并完成付费（如适用）后，获得更高使用层级与 API 权限的用户。
  • 高级模型：因成本、风险或供应商限制而需更严格审核与限制的模型。
  • API：指通过编程方式（如 HTTP/SDK）调用平台提供的接口进行模型交互的能力。
  • 服务范围/支持范围：指平台承诺提供的可支持事项；未明确包含在服务范围的事项，平台不承担支持义务。
  • 滥用：包括但不限于突破限流、批量生成、外部代调用、规避风控、恶意负载等。

第二章 服务范围与责任边界

灵息 AI 为用户提供第三方模型的技术接入与聚合体验，不拥有、不控制、不干预、不修改第三方模型的算法与逻辑。本平台仅提供连接、编排、调度与基础风控能力。

  • 所有模型输出均由第三方模型自动生成，与本平台无关。
  • 本平台不对输出内容的准确性、完整性、合法性、可执行性做出任何承诺或保证。
  • 用户应自行核验并对使用结果承担全部责任。
  • 平台可基于安全、合规、成本与服务质量对接入的模型、功能与配额进行调整或下线。

2.1 API 权限不等于支持义务

Plus 用户拥有 API 权限，但 API 属于“自助能力”，平台仅提供接口的基础可用性与必要的风控限制。API 不构成平台的客服支持范围，包括但不限于：

  • 任何第三方应用程序是否支持 API、是否兼容本平台接口。
  • 应用程序配置问题（参数、代理、证书、网络环境、环境变量等）。
  • 用户自行开发/集成 API 的开发问题（代码错误、鉴权错误、SDK/库兼容、并发架构等）。
  • 用户因误用 API 导致的成本、用量、封禁、业务损失等后果。

2.2 “平台不可用”与“模型表现不好”的边界

  • 平台不可用：指登录、会话、路由、Open WebUI 端交互无法正常使用等平台侧故障，属于可支持范围（见第二十章）。
  • 模型表现：指幻觉、编造、错误回答、低质量内容、跑题等输出质量问题，属于第三方模型行为，不在支持范围。

第三章 账户注册与管理

3.1 信息真实性

注册信息需真实、准确、完整，并及时更新。因信息不实导致的后果由用户自行承担。

3.2 账户数量限制

每位自然人/法人最多可拥有 5 个灵息账户。如发现恶意多开、批量注册或规避设备指纹，将直接封禁账号、设备与 IP。

3.3 账户安全

用户对其账户、令牌、密码等安全要素承担全部保管责任。因保管不当导致的损失由用户自行承担。

3.4 身份校验

为保障合规与风控，平台可能要求进行邮箱/手机号验证、二步验证、实名/企业认证（如适用）。未按要求完成者，功能可能受限或无法使用。

3.5 风控与异常行为

为保护平台稳定性与全体用户公平性，平台可能对异常登录、异常用量、疑似共享/代调用、疑似自动化滥用等行为进行验证、限流、冻结或封禁。用户应配合完成必要验证，否则服务可能受限。

第四章 用户等级与权限（含 Plus）

4.1 普通用户（免费版）

  • 可在平台界面内使用部分模型与功能。
  • 不提供 API 权限：无 API Key、无编程调用、无第三方集成。
  • 适用公平使用策略与动态限额，具体限流/配额参数不公开。
  • 支持渠道以社区/公告为主，属于有限支持（见第二十章）。

4.2 Plus 用户（登记/审核）

  • 拥有更高的使用层级：更高并发、配额、上下文长度与优先队列（以页面公示为准）。
  • 开放 API 权限：可申请并管理 API Key，进行编程调用。
  • 重要说明：Plus 提供的是“权限与资源等级”，不代表平台对 API 集成、应用配置、开发调试提供支持；API 不在支持范围（见第二章与第二十章）。
  • 若持续滥用或大批量外部代调用，将被限制或封禁，不因 Plus 身份而豁免。

4.3 Plus 用户登记流程

  1. 在账户设置页提交登记申请，说明合理、真实、明确的使用场景与需求规模。
  2. 阅读并确认同意 API 与高级模型使用规范。
  3. 如涉及付费，完成支付与开通；如需企业资质或实名，按指引完成验证。
  4. 平台进行合规与风控审核；平台保留无条件通过或拒绝的权利。

4.4 升降级与资格撤销

平台可基于风险、成本、合规或服务质量原因，调整用户等级、权限与配额；对于滥用、造假、共享账号等行为，平台可随时撤销 Plus 权限并采取封禁措施。

4.5 权限的个体化与差异

由于供应商政策、成本波动、风险等级、区域与合规要求不同，Plus 用户之间可能存在可用模型、配额与功能差异。平台以控制台展示与公告为准。

第五章 API 使用与密钥管理

5.1 免费版不提供 API

免费版不提供任何 API 权限。免费用户不得通过任何形式模拟、抓取、代理或中转以实现编程调用。

5.2 Plus API 权限（自助能力）

  • API Key 为个人或组织专属，不得共享、出租、转借、售卖或用于外部代调用。
  • 仅可用于申请时承诺的自有业务场景；变更用途须重新登记。
  • 严禁将本平台 API 封装为对外服务、SDK、代理节点或竞品能力。
  • API 不在平台支持范围内：平台不负责用户侧的应用适配、参数配置、开发调试、性能调优与错误排查。

5.3 配额、并发与限流

  • 平台采用动态限额策略；为保障风控，不公开具体阈值与参数。
  • 配额、QPS、并发与速率限制以控制台/公告为准，平台可动态调整。
  • 禁止通过多账户、并行队列、代理池等方式规避限流。
  • 一旦判定为滥用，平台可立即限制或封禁账号、设备与 IP。

5.4 密钥安全与合规

  • 用户应安全存储 API Key，密钥泄露后果自负。
  • 应在后端安全环境调用，避免在前端或客户端硬编码暴露。
  • 应遵守第三方供应商政策、出口管制与当地法律。

5.5 SDK、代理与第三方集成

对第三方 SDK/代理/自动化工具的使用，需同时遵守本协议及第三方条款；如对稳定性或合规构成风险，平台可限制或下线相关集成。用户因使用第三方工具导致的错误、封禁、数据泄露与损失由用户自行承担。

第六章 高级模型使用规定（重点强化）

6.1 模型分级机制

模型分为普通模型与高级模型。高级模型因资源成本、供应商限制或滥用风险更高，需额外审核或邀测方可启用。

6.2 申请要求

  • 提交合理、真实、明确的使用理由与规模预估。
  • 承诺不滥用、不共享、不出租、不转借账号或密钥。
  • 承诺不进行批量生成、商业外包、竞品封装或外部代调用。
  • 遵守所有模型供应商的使用政策与限制。

6.3 管理员审核与撤销权

  • 管理员有权无条件决定是否通过审核，无需提供理由。
  • 管理员有权随时撤销高级模型权限，无需提前通知，也无须说明理由。
  • 出现滥用、造假、高频攻击、共享账号等行为，平台可永久封禁账号、设备与 IP。

6.4 使用限制

高级模型禁止用于商业对外服务、二次分发、批量任务或任何形式的外部提供服务。违规将立即封禁。

第七章 用户行为规范与禁止事项

7.1 内容红线

禁止生成或传播违法、暴力、色情、侵权、诈骗、谣言、仇恨与歧视等内容。

7.2 技术与访问滥用

  • 禁止破解、攻击、逆向工程或干扰平台系统与风控。
  • 禁止使用脚本、爬虫、自动化工具规避限流与配额。
  • 禁止外部代调用、流量中转、分流与二次封装为对外产品。

7.3 QPS/并发限制

平台基于公平使用原则设置 QPS、并发与配额；平台采用动态限额且不公开具体参数。一旦发现规避行为或异常负载，将立即采取限制、冻结或封禁。

7.4 账号共享与外部代用

严禁共享账号、共享 API Key、将能力提供给非登记主体使用或对外收费提供服务。平台有权对相关账号、设备与 IP 采取即时制裁，并可能追究法律责任。

第八章 数据、隐私与安全

  • 平台可能为计费、风控、服务改进而收集必要的日志与元数据（如时间、路径、用量、状态、设备与网络特征）。
  • 平台采取合理技术与管理措施保护数据安全，但不对绝对安全作出保证。
  • 用户应避免上传敏感信息；如需处理敏感数据，应确保具备合法授权与合规基础。
  • 日志与数据保留周期以隐私政策或控制台公告为准；用户可按指引申请删除（法律法规另有规定除外）。

8.1 数据最小化与风险自担

平台建议用户遵循“最小化输入”原则：不输入不必要的个人信息、机密信息、身份证件、账号密码、私钥与未公开商业资料。用户因自行上传或分享导致的泄露风险由用户自行承担。

第九章 知识产权与内容政策

  • 用户对其上传的内容拥有合法权利，并对由此引发的争议与风险负责。
  • 模型输出内容的权利与约束，须同步遵守对应第三方供应商的政策与当地法律。
  • 平台或其关联方拥有网站/客户端的界面、代码与文档等的知识产权与权利保留。

9.1 侵权投诉与处置

如平台收到合理的侵权投诉或合规要求，平台有权在不预先通知的情况下采取必要措施，包括但不限于内容限制、模型禁用、账号限制或配合取证。用户应依法自行解决由其内容引发的纠纷。

第十章 费用、计费与退款（Plus）

  • Plus 为增值服务，可能按订阅、用量或混合模式计费，以页面公示与订单为准。
  • 已支付费用原则上不予退还，除非法律规定或平台另行明示的退款政策。
  • 如用户存在滥用、违约或违规，平台可暂停或终止服务且不予退款。
  • 如涉及自动续费，续费规则、发票与税费以支付页面与账单为准。

10.1 价格与权益调整

平台可因供应商成本、汇率、合规与运营策略调整价格与权益结构。已生效订单按订单约定执行；新价格适用于调整后的新订单或续费订单（如适用）。

第十一章 服务可用性与变更

  • 平台不对服务的持续可用性、实时性或无故障运行作出保证。
  • 平台可因维护、升级、供应商变更、法律合规要求而调整或中断服务。
  • 为应对滥用风险与弹性资源调度，平台实行动态限额并可随时调整；不公开具体参数。
  • 重大变更将通过站内通知、邮件或公告进行提示。

11.1 不可抗力

因不可抗力或第三方原因（包括但不限于云服务故障、上游模型供应商中断、网络运营商故障、政策变化、自然灾害等）导致的服务中断或异常，平台在法律允许范围内免责或限责。

第十二章 免责声明与风险提示

  • 模型输出可能包含错误、幻觉、不完整或不准确信息。
  • 生成的代码可能存在逻辑错误、漏洞或无法编译/执行。
  • 平台不保证任何输出内容的正确性、完整性或适用性，相关风险由用户自行承担。
  • 对因使用或无法使用本服务导致的直接或间接损失，平台在法律允许范围内免责或限责。
  • API 不在支持范围内：用户因 API 集成/开发/配置导致的问题与损失由用户自行承担。

12.1 风险提示

  • 请勿将模型输出作为医疗、法律、投资等高风险决策的唯一依据。
  • 涉及敏感业务、合规审核、对外发布内容时，建议进行人工复核与专业审查。
  • 使用自动化或批量任务前，请评估成本、限流与封禁风险。

第十三章 第三方政策与合规

用户必须遵守第三方模型供应商的政策，包括但不限于 OpenAI、Google DeepMind、Anthropic、DeepSeek、阿里云、百度、智谱 AI 等。若用户违反供应商政策，本平台可按对方要求对用户采取警告、限制或永久封禁措施。

第十四章 出口管制与地域限制

用户应遵守适用的出口管制、制裁与贸易合规要求。对于受限地区、个人或实体，平台可拒绝提供服务或采取限制措施。

第十五章 未成年人与教育用途

未成年人应在监护人指导下使用平台并遵循本协议。用于教育场景时，应由合规主体负责内容审核与使用规范。平台不对教育成果、作业正确率、考试成绩等作出任何承诺。

第十六章 反馈、开源与社区

  • 用户可向平台提交反馈与建议；平台有权决定是否采纳，且不因采纳而承担付费义务。
  • 如平台提供开源组件，需遵守对应开源许可证。
  • 社区交流应遵守基本礼仪与法律法规；平台可对影响社区秩序的行为采取禁言、移除或封禁措施。

第十七章 终止、违约与制裁措施

  • 平台可基于合规与风控，随时对违规/高风险账户采取限制、冻结或终止服务。
  • 对严重滥用（含 Plus），平台可永久封禁账号、设备与 IP，并保留追究法律责任的权利。
  • 用户可按流程申请注销账户；法律法规或账务未结清的情形下，平台可暂缓处理。

17.1 违约成本

用户因违反本协议、供应商政策或法律法规导致的处罚、赔偿、律师费、调查成本与平台损失，由用户自行承担。平台有权在法律允许范围内向用户追偿。

第十八章 通知、更新与解释权

  • 本协议可能随时更新；新版发布后立即生效，用户继续使用即视为同意。
  • 平台可通过站内信、邮件、弹窗或公告进行通知。
  • 本协议条款标题仅为方便阅读，不影响条款之解释。

18.1 分条可分割

若本协议任何条款被认定为无效或不可执行，不影响其他条款的效力与可执行性；双方应以合法有效且最接近原意的条款替代。

第十九章 争议解决与法律适用

本协议之订立、效力、解释、履行与争议解决，适用平台运营地法律。双方应首先友好协商；协商不成的，任一方可向平台运营地有管辖权的人民法院提起诉讼。

第二十章 客服与支持范围

20.1 服务渠道与等级

  • 免费用户：仅提供有限的社区支持与公告指引（例如常见问题、已知故障通告、社区群内交流）。平台不承诺响应时效与处理优先级。
  • 付费/Plus 用户：可在站内通过私信/反馈功能提交问题与建议，享有更优先的支持队列，但平台仍不提供 SLA/固定响应时间承诺。

20.2 平台可支持事项（仅限 Open WebUI 端）

平台仅对以下类型问题提供合理范围内的支持（以 Open WebUI 端为准）：

  • 登录、注册、验证、账户异常（在合规允许范围内）。
  • 站内会话交互不可用、页面功能故障、路由/模型列表异常、基础可用性问题。
  • Plus 开通、订单与计费展示类问题（不含第三方支付机构自身故障）。
  • 平台公告中声明可受理的其他事项。

20.3 明确不支持事项

  • AI 自身问题不支持：幻觉、瞎编、错误信息、低质量输出、跑题、风格不符合预期等。
